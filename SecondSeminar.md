# Second Seminar

## Creating branches
git branch branch_name - создание новой ветки
git checkout -b branch_name - создание новой ветки с одновременным переходом на нее
## Merging branches
git merge branch_name - слияние веток
## Conflicts
Конфликты возникают когда затронуто общее пространство
## Работа с удаленным репозиторием
* Клонировать удаленный репозиторий - __git clone <адрес_репозитория>__
* Создаем ветку в которой будем работать и сохраняем изменения
* Переключаемся в master

**Перед тем как сливать новый merge, обновляем локальную ветку master**

* Обновляем локальную ветку с сервера -
__git pull origin master__
* делаем merge с веткой в которой мы работали
* Сливаем изменения в удаленный репозиторий - __git push origin master__
* Разрешаем конфликт
